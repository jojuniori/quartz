# 微信读书官方Skill

![[知识/AI/_assets/weread-skills/cover.jpg]]

## 一句话总结

微信读书官方 Skill 可以连接微信读书账号，让 AI 查询书架、阅读统计、笔记、划线，以及搜索书籍；它更像“阅读数据接口”，不是替你直接读完一本书的工具。

## 核心信息

| 项目 | 内容 |
| --- | --- |
| 名称 | 微信读书官方 Skill |
| 类型 | AI Skill / 阅读数据连接器 |
| Skill 下载 | https://cdn.weread.qq.com/skills/weread-skills.zip |
| API Key | https://weread.qq.com/r/weread-skills |
| 适配场景 | OpenClaw / Hermes 等 Agent 环境 |

## 安装方式

在支持 Skill 的 Agent 环境里发送或执行：

```text
下载 https://cdn.weread.qq.com/skills/weread-skills.zip 安装 skill
```

然后前往微信读书官方页面扫码登录并获取 API Key：

```text
https://weread.qq.com/r/weread-skills
```

配置完成后，就可以让 AI 读取账号内的相关阅读数据。

## 主要功能

- **查书架**：让 AI 知道你当前书架里有哪些书。
- **查阅读统计**：读取阅读时长、阅读进度等统计信息。
- **查笔记和划线**：把已有批注、摘录、划线作为 AI 分析材料。
- **搜索书籍**：辅助查找微信读书里的书籍。

## 它不能做什么

这个 Skill 的重点不是“帮你读书”，而是把你已有的阅读数据和微信读书账号能力暴露给 AI。

换句话说，它更适合：

- 整理你已经读过的书。
- 汇总你的划线和笔记。
- 分析你的阅读偏好。
- 基于书架和笔记生成复盘、书单或主题总结。

不适合期待它直接替你完整阅读一本新书并输出高质量理解。

## 我的判断

这个 Skill 对个人知识管理很有价值：微信读书里沉淀的划线、笔记、书架，终于可以比较自然地进入 Agent 工作流。

比较值得尝试的工作流：

- 每周总结最近阅读和划线。
- 从某本书的笔记里提炼主题卡片。
- 根据书架生成“待读 / 已读 / 主题阅读”列表。
- 把微信读书笔记迁移或整理进 Obsidian。

需要注意的点：

- API Key 属于账号访问凭据，配置时注意隐私和保管。
- AI 能看到的内容取决于 Skill 暴露的权限范围。
- 如果用于自动同步笔记，要先确认重复写入和格式控制。
