# Dokobot-Agent网页浏览工具

## 一句话总结

Dokobot 是一个给 AI Agent 使用的网页浏览工具，目标是让 Agent 能更稳定地打开网页、读取内容、截图、操作页面，并把结果返回给用户或上层自动化流程。

## 核心信息

| 项目 | 内容 |
| --- | --- |
| 名称 | Dokobot |
| 类型 | AI Agent 网页浏览 / 浏览器自动化工具 |
| 官网 | https://dokobot.com |
| 适合场景 | 网页读取、页面截图、表单操作、网页任务自动化 |

## 主要能力

- **网页浏览**：替 Agent 打开网页并读取页面内容。
- **视觉理解**：可结合截图让 Agent 观察网页状态。
- **页面操作**：适合执行点击、输入、跳转等交互任务。
- **自动化任务**：可作为网页任务执行层，配合上层 Agent 完成更长流程。

## 适合场景

- 抓取页面信息并总结。
- 检查网页 UI 或流程是否正常。
- 自动填写表单、执行重复网页操作。
- 需要让 Agent 与真实网页环境交互的任务。
- 替代只读网页解析器，处理需要登录、点击、动态渲染的页面。

## 我的判断

Dokobot 的价值在于把“网页”变成 Agent 可操作的环境。相比单纯 HTTP 抓取，它更接近真实用户浏览器行为，适合处理动态页面和需要交互的任务。

需要关注的点：

- 是否支持登录态、Cookie 和持久会话。
- 对复杂网页的稳定性如何。
- 截图、DOM、文本三类信息能否一起提供给 Agent。
- 自动化操作是否有速率限制和安全边界。
